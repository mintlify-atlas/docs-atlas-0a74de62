This page documents all CVars registered by the game module (`qagame.qvm`) in `code/game/g_main.c`. CVars can be set on the server console, in `server.cfg`, or passed on the command line as `+set cvarname value`.

## CVar flags

| Flag | Value | Description |
|---|---|---|
| `CVAR_ARCHIVE` | `1` | Saved to `q3config.cfg` on exit |
| `CVAR_USERINFO` | `2` | Sent to server in the userinfo string on connect or change |
| `CVAR_SERVERINFO` | `4` | Included in the server info response visible to server browsers |
| `CVAR_SYSTEMINFO` | `8` | Broadcast to all clients as system info |
| `CVAR_LATCH` | `32` | Change takes effect only after the next map load |
| `CVAR_ROM` | `64` | Read-only; cannot be set by users |
| `CVAR_NORESTART` | `1024` | Preserved across `cvar_restart` |

## Game CVars

### Game type and match settings

| CVar | Default | Flags | Description |
|---|---|---|---|
| `g_gametype` | `0` | SERVERINFO, USERINFO, LATCH | Game type: 0=FFA, 1=Tournament, 2=Single Player, 3=Team DM, 4=CTF |
| `fraglimit` | `20` | SERVERINFO, ARCHIVE, NORESTART | Number of frags to end the match. Announced to clients when changed. |
| `timelimit` | `0` | SERVERINFO, ARCHIVE, NORESTART | Time limit in minutes. 0 = no time limit. Announced to clients when changed. |
| `capturelimit` | `8` | SERVERINFO, ARCHIVE, NORESTART | Flag captures required to win CTF. Announced to clients when changed. |
| `dmflags` | `0` | SERVERINFO, ARCHIVE | Bitmask of deathmatch flags. `8`=no falling damage, `16`=fixed FOV, `32`=no footsteps. Announced when changed. |

| CVar | Default | Flags | Description |
|---|---|---|---|

| `g_maxGameClients` | `0` | SERVERINFO, LATCH, ARCHIVE | Maximum number of clients who can be in the game (non-spectating). 0 = no limit. |

## Registering custom CVars

To add a CVar in your mod, add an entry to `gameCvarTable[]` in `g_main.c`:

```c
// In g_main.c — add to the vmCvar_t declarations:
vmCvar_t  g_myOption;

// Add to gameCvarTable[]:
{ &g_myOption, "g_myOption", "1", CVAR_ARCHIVE, 0, qtrue },
```

The `trackChange` field (`qtrue` in the last column) causes the game to broadcast a message to all clients when the CVar changes. CVars registered here are automatically updated each frame by `G_UpdateCvars`.
