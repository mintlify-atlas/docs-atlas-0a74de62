---
title: "Building from Source"
description: "How to compile the Quake III Arena GPL source release on Windows, Linux, and macOS."
---

The Quake III Arena GPL release ships project files and build scripts for all three supported platforms. The code compiles to a standalone `quake3` (or `quake3.exe`) binary plus a set of QVM bytecode files for the game, cgame, and UI modules.

<Warning>
This is the original 2005 GPL release. The build toolchain targets compilers from that era (MSVC 7.1, GCC 2.95). Modern compiler versions require community-maintained patches and are not covered here.
</Warning>

## Prerequisites

<Tabs>
  <Tab title="Windows (MSVC)">
    - **Visual C++ 2003 (VC7 / Visual Studio .NET 2003)** — the solution files target this toolset.
    - **NASM** — for the x86 assembly optimisation files (`vm_x86.c` calls into hand-written asm stubs).
    - **Pre-built QVM tool binaries** — `lcc.exe`, `q3cpp.exe`, `q3rcc.exe`, and `q3asm.exe`. Pre-compiled copies are provided under `lcc/bin/` and `code/win32/mod-sdk-setup/bin/`. These must be on your `PATH` before running the QVM batch files.
  </Tab>
  <Tab title="Linux (GCC)">
    - **GCC 2.95** and **G++ 2.95** — the build scripts hard-require this version.
    - **NASM** — for x86 JIT assembly.
    - **X11 development headers**, including the *X Direct Graphics Access* (`XF86DGA`) and *X Video Mode* (`XF86VidMode`) extension headers.
    - **Perl** — required by the `cons` build system included in the tree.
  </Tab>
  <Tab title="macOS (Xcode)">
    - **Mac OS X with Xcode** — a `.pbproj` project file is provided for the classic Project Builder / early Xcode toolchain.
    - No additional command-line tools are strictly required beyond Xcode.
  </Tab>
</Tabs>

## Getting the Source

The GPL source was released as a tarball. If you are working from a git mirror:

```bash
git clone <repository-url> quake3-gpl
cd quake3-gpl
```

The top-level layout after extraction:

```
code/        Engine source (renderer, game, OS layer, etc.)
lcc/         Retargetable C compiler (produces QVM assembly)
q3asm/       Assembly → QVM bytecode compiler
q3map/       Map compiler (.map → .bsp)
q3radiant/   Q3Radiant map editor (build 200f)
common/      Shared support code for radiant / tools
libs/        Third-party library support (pak, jpeg, etc.)
```

## Build Steps

<Tabs>
  <Tab title="Windows">
    <Steps>
      <Step title="Open the solution">
        Open `code/quake3.sln` in Visual C++ 2003 (VC7).

        A second solution, `q3radiant/Radiant.sln`, builds the map editor separately.
      </Step>
      <Step title="Build the engine">
        Select **Build → Build Solution** (or press `F7`). This compiles the engine and produces `quake3.exe`.
      </Step>
      <Step title="Add QVM tools to PATH">
        The game logic runs inside the QVM virtual machine. To compile `.qvm` files you need the QVM toolchain on your `PATH`:

        ```bat
set PATH=%PATH%;C:\path\to\quake3-gpl\lcc\bin
        ```

        Pre-compiled binaries are also provided in `code/win32/mod-sdk-setup/bin/`.
      </Step>
      <Step title="Compile the QVM modules">
        Each game module has a batch file that compiles the C sources through `lcc` → `q3asm` and produces a `.qvm`:

        ```bat
cd code\game
build.bat

cd ..\.\cgame
build.bat

cd ..\.\q3_ui
build.bat
        ```

        The resulting `.qvm` files are placed in `baseq3/vm/`.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Linux">
    <Steps>
      <Step title="Install dependencies">
        ```bash
# Debian / Ubuntu example
apt-get install gcc-2.95 g++-2.95 nasm \
  libxxf86dga-dev libxxf86vm-dev
        ```
      </Step>
      <Step title="Run the cons build">
        The build system is `cons` (a Perl-based Make alternative). A copy of the build script is bundled in the tree:

        ```bash
cd code
./unix/cons -- gcc=gcc-2.95 g++=g++-2.95
        ```

        Pass `-- target=release` for an optimised build, or omit it for the debug default.
      </Step>
      <Step title="Inspect build output">
        The compiled binary and shared objects are placed in `code/` and in platform-specific subdirectories. The engine executable is typically named `quake3.x86`.
      </Step>
    </Steps>
  </Tab>
  <Tab title="macOS">
    <Steps>
      <Step title="Open the project">
        Open `code/macosx/Quake3.pbproj` in Xcode (or the classic Project Builder).
      </Step>
      <Step title="Build">
        Select **Build → Build** from the menu. The project is configured for the PowerPC target but also compiles for early x86 Macs with minor modifications.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Artifacts Produced

| Artifact | Description |
|---|---|
| `quake3` / `quake3.exe` | Main engine executable |
| `baseq3/vm/qagame.qvm` | Server-side game logic (runs in QVM) |
| `baseq3/vm/cgame.qvm` | Client-side game logic (runs in QVM) |
| `baseq3/vm/ui.qvm` | Menu / UI system (runs in QVM) |
| `quake3.x86` (Linux) | Engine binary, x86 Linux |

<Note>
The QVM files require the base game data (`pak0.pk3` through `pak8.pk3`) from a legitimate Quake III Arena installation to run. The GPL release is source code only — no game data is included.
</Note>
